# 手牌评分规格

## 1. 目的
对手牌强度进行量化评估，辅助智能体决策。评分结果可用于：
- 本地开局策略的选择
- 传递给 AI 模型作为额外上下文
- 调试输出中的手牌质量展示

## 2. 评分框架

- **总分范围**：0 - 100
- **维度权重**：
  - 结构分（40%）：手牌整齐度、散牌数量、长套连贯性
  - 控制力分（30%）：炸弹/同花顺/天王炸、级牌/王/A的数量
  - 潜力分（30%）：逢人配、牌型多样性

## 3. 数据来源
评分所需数据完全来自 `observation.my_info` 和引擎返回的原始 `legal_actions`（非剪枝后），不直接访问引擎内部状态。

## 4. 结构分计算（满分40）

### 4.1 基础散牌分（根据 `remaining_single_card_count`）
| 散牌数量 | 基础分 |
|----------|--------|
| 0 - 1    | 40     |
| 2 - 3    | 35     |
| 4 - 5    | 28     |
| 6 - 7    | 20     |
| 8+       | 10     |

### 4.2 长套奖励（附加）
| 牌型    | 加分条件                                  | 每项加分 | 上限 |
|---------|-------------------------------------------|----------|------|
| 钢板    | 存在至少一个钢板动作                      | +5       | +5   |
| 顺子    | 每多一个额外顺子（固定5张）               | +2       | +6   |
| 连对    | 每多一个额外连对                          | +2       | +4   |
| 三带二  | 每多一个额外三带二                        | +1       | +3   |

> 最终结构分 = min(40, 基础散牌分 + 长套奖励总和)

## 5. 控制力分计算（满分30）

### 5.1 炸弹评分（上限15）
| 炸弹类型           | 每项加分 |
|--------------------|----------|
| 4张炸弹            | +3       |
| 5张炸弹            | +5       |
| 6张及以上炸弹      | +8       |
| 同花顺             | +8       |
| 天王炸             | +10      |

累计值按比例缩放至15分（若累计≤15则直接取，超过则取15）。

### 5.2 控制牌评分（上限15）
| 牌型       | 每张加分 |
|------------|----------|
| A          | +1       |
| 级牌（非逢人配） | +1.5 |
| 小王       | +3       |
| 大王       | +4       |

逢人配（红桃级牌）在此只算作级牌，不额外加分。  
累计值封顶15分。

> 控制力分 = min(15, 炸弹分) + min(15, 控制牌分)

## 6. 潜力分计算（满分30）

### 6.1 逢人配加成（上限10）
- 有 1 张逢人配：+10
- 0 张：+0

### 6.2 牌型多样性（上限20）
（统计 `legal_actions` 中出现的不同 `declared_pattern` 种类数）
| 不同牌型种类数 | 得分 |
|----------------|------|
| 5 种及以上     | 20   |
| 4 种           | 15   |
| 3 种           | 10   |
| 2 种           | 5    |
| 1 种           | 0    |

> 潜力分 = 逢人配加成 + 多样性分

## 7. 总分与标签
- **总分** = 结构分 + 控制力分 + 潜力分（若 >100 则截断为 100，若 <0 则为 0）
- **手牌强度标签**：根据总分划分
  - ≥ 80：极强
  - 60 - 79：较强
  - 40 - 59：中等
  - 20 - 39：偏弱
  - < 20：极弱
- **评语生成**：根据各分项突出特点组合成简短描述，例如“手牌结构优良，炸弹充足，控制力强，逢人配灵活，潜力大”。

## 8. 开关控制
手牌评分功能默认关闭，避免非必要性能消耗。通过环境变量 `HAND_EVALUATION_ENABLED` 控制：
- `true` / `1`：启用评分，并在 `--show-thinking` 时输出评分信息
- `false` / `0`：关闭评分，相关代码不执行

## 9. 更新规则
当需要调整评分权重、阈值或加分值时，应先修改本文件，再根据本文件的变更内容调整 `agents/hand_evaluator.py` 的实现。未经文档变更，代码的评分行为不得偏离本文件规定。
