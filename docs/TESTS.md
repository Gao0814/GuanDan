# TESTS

## 1. 测试目标
验证第一阶段系统满足以下要求：

- 掼蛋引擎规则正确
- 状态转移一致

- AI 不出非法牌
- AI 出牌不表现为明显纯随机乱出

- DeepSeek API 接入后不会替代规则真值
- 开发者可通过调试输出验证每一步

## 4. 基础 AI 行为测试

### A1：固定局面下动作稳定性
在相同输入局面下，多次运行基础 AI。  
测试口径：
- 同一固定局面连续运行 10 次
- 若该局面无并列合理动作，则 10 次输出必须完全一致
- 若存在并列合理动作，则输出必须始终落在预定义“合理动作集合”内
- 不允许输出落到合理动作集合外

### A2：不无意义提前打出高价值牌
构造一个同时存在普通低价值动作与明显高价值控制牌的局面。  
测试口径：
- 若存在至少 1 个低价值且同样合法的基础动作，AI 默认不应优先选择高价值控制牌
- 若选择高价值控制牌，则该测试样例必须显式声明触发理由
- 无显式触发理由时，选择高价值牌判定为失败

### A3：不无意义拆解基础组合
构造一个存在明显基础组合保留价值的局面。  
测试口径：
- 若存在不拆组也能完成当前合法出牌的动作，则 AI 默认不应选择拆组动作
- 若选择拆组动作，则该测试样例必须显式声明允许拆组的理由
- 无显式理由时，拆组判定为失败

### A4：固定局面中优先明显更合理动作
构造一个普通玩家也容易判断的简单局面。  
测试口径：
- 每个样例预先标记“明显更合理动作集合”与“明显更差动作集合”
- AI 输出必须命中前者，不得命中后者

### A5：新增牌型加入后经验排序仍然稳定
在引入更多基础牌型后，对固定测试局面运行 AI。  
预期：
- 不因新牌型接入导致旧经验规则完全失效
- 仍能保持最小经验一致性
- 不退化为随机或明显失真出牌

## 5. 集成测试

### I1：4 AI 完整自动对局
启动 4 AI 一局完整对局。  
预期：
- 对局结束
- 无异常
- 无非法动作进入状态更新

### I2：多局自动对局稳定性
运行多局 4 AI 自动对局。  
预期：
- 无死循环
- 无状态污染
- 非法动作率为 0

### I3：全信息调试输出完整
运行调试模式。  
预期至少包含以下字段：

- `step_id`
- `current_player_id`
- `all_hands`
- `table_constraint`
- `legal_actions`
- `chosen_action`
- `decision_basis`
- `state_before`
- `state_after`
- `remaining_hand_counts`
- `round_ended`
- `game_over`
- `winner`（若已结束）

并且：
- `state_before` 与 `state_after` 至少应包含当前行动玩家、桌面约束、各玩家剩余手牌数、最近成功出牌玩家、当前是否处于领出阶段或跟牌阶段

### I4：补全规则后的 4 AI 自动对局稳定性
在加入大王、小王和新增基础牌型后，重新运行多局自动对局。  
预期：
- 对局仍可结束
- 非法动作率为 0
- 旧规则行为不被破坏
- 新规则不引入明显状态污染

### I5：API 未接入时主链路仍稳定
在未启用 DeepSeek API 的情况下运行 4 AI 自动对局。  
预期：
- 系统仍能正常运行
- 规则真值链路不受影响

### I6：API 接入后主链路仍受规则引擎约束
在启用 DeepSeek API 后运行测试局。  
预期：
- 模型支持不改变合法动作边界
- 模型输出不替代引擎裁决
- 冲突知识不会进入规则真值链路

## 6. RAG 与知识支持测试

### R1：合法动作生成与状态机一致
每个合法动作都能被状态机接受。

### R2：非法动作不会污染状态
提交非法动作后，状态保持不变。

### R3：RAG 不混入其他游戏规则
检索内容不应出现其他纸牌游戏规则条目。

### R4：日志可用于复盘
动作日志足以复现关键过程。

### R5：基础 AI 不退化为纯随机
在固定测试局面集上回归运行。  
预期：
- 行为不退化为纯随机采样
- 不退化为明显不合理出牌

### R6：rejected_conflict 不进入可执行依据
当检索命中冲突知识时：  
预期：
- 可被记录为 rejected
- 不进入规则性可执行依据
- 不替代引擎真值

### R7：知识库正文与当前实现一致
规则库正文、经验库正文与当前已实现规则范围保持一致。  
预期：
- 当前未实现规则不能被写成“当前可执行规则”
- 当前已实现规则必须能在知识库中找到一致表述

### R8：DeepSeek 接入失败不破坏规则主链路
模拟 API 不可用或调用失败。  
预期：
- 系统仍可依靠本地规则引擎运行
- 不会导致非法动作进入状态更新
