# PLAN

## 1. 当前阶段总体目标

当前阶段的总体目标是：

> 在现有项目基础上，完成一个支持完整单局核心规则的掼蛋引擎，并在其上叠加一个只从合法动作集合中选择 `action_id` 的 AI 决策层。

当前阶段希望最终具备：

- 单局掼蛋引擎闭环
- 4 AI 自动完整对局
- 开发者全信息调试
- 基础规则型 / 经验型 AI
- RAG 规则库与经验库接入
- 为 DeepSeek API 留出清晰接入边界
- 基础测试与日志
- 后续人类接入的基础接口预留

---

## 2. 当前阶段主线判断

当前阶段的唯一主线是：

- 单局核心规则闭环
- 引擎与 AI 职责边界收敛
- observation / action / step 接口契约收敛
- 测试口径与规则口径统一

因此，当前阶段不再以以下内容为主推进目标：

- 多局升级赛
- 打到 A 的长期比赛制
- 复杂贡还 / 抗贡比赛制
- 完整地方规则百科
- 强化学习 / MCTS / 自博弈训练

## 4. 当前阶段实施原则

后续工作必须遵守以下原则：

1. 先改文档，再补测试，再改代码
2. 先保证单局核心规则稳定，再接模型支持
3. 规则引擎与 AI 决策必须分离
4. 任何新增规则都必须同步补：
   - 牌型识别
   - 合法动作生成
   - 压牌规则
   - 状态转移
   - 测试
   - 知识库正文
5. 采用“先测后码、最小变更”策略
6. 不顺手扩张到比赛层或第二阶段

## 7. 当前风险点

1. 当前模型决策稳定性已较此前改善，但仍可能在复杂跟牌场景中低估跨型压制收益
2. 上下文复杂度继续上升后，手牌评分、记牌信息和理牌摘要可能增加 prompt 长度并影响响应速度
3. 理牌 / 剪枝若过度压缩，可能损伤关键动作可见性，需持续验证保留策略
4. 若后续规则微调（如顺子窗口边界），需同步更新 RAG 知识库和测试用例，否则会出现口径不一致

---

## 8. 当前建议顺序

建议严格按以下顺序推进：

1. 改文档
2. 补测试
3. 改引擎
4. 收口 AI / RAG / CLI
5. 接 DeepSeek API
6. 做完整回归

未经文档与核心测试对齐，不进入大规模代码实现。
